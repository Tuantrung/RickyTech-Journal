## What this project is

A personal tech blog — **RickyTech Journal** — built with [Hugo](https://gohugo.io) and the [PaperMod](https://github.com/adityatelange/hugo-PaperMod) theme. It deploys automatically to GitHub Pages on every push to `master`.

## Common commands

```bash
# After cloning — initialise the PaperMod theme submodule
git submodule update --init --recursive

# Local dev server (live reload)
hugo server -D

# Build site (output goes to public/)
hugo --gc --minify

# Create a new post
hugo new posts/my-post-title.md
```

New posts are created as drafts (`draft = true` in frontmatter). Remove or set to `false` to publish.

## Architecture

| Path | Purpose |
|------|---------|
| `content/posts/` | Blog posts as Markdown with TOML frontmatter |
| `hugo.toml` | Site config: theme, params, menus, search, analytics |
| `themes/PaperMod/` | Git submodule — do not edit directly |
| `layouts/` | Local overrides of PaperMod templates |
| `static/admin/` | Decap CMS UI (accessible at `/admin/`) |
| `public/` | Build output — generated, do not hand-edit |
| `.github/workflows/hugo.yml` | CI: builds and deploys to GitHub Pages |

## Key conventions

- `content/search.md` and `content/archives.md` are required for the Search and Archives nav items — do not delete them. The JSON output (`[outputs] home = ["HTML","RSS","JSON"]` in `hugo.toml`) powers the client-side search; removing it breaks search.
- **Frontmatter format is TOML** (`+++` delimiters), not YAML.
- **Theme customisation** goes in `layouts/` (local overrides), not in `themes/PaperMod/`. The only existing override is `layouts/taxonomy/terms.html` which renders the tags listing.
- The CI workflow uses Hugo **extended** v0.151.0. If a feature requires extended (Sass, image processing), it works in production but you need `hugo_extended` locally too.
- `static/admin/config.yml` points to the `tuantrung97/RickyTech-Journal` GitHub repo — update the `repo:` field if the repo is renamed or forked.
- The `public/` directory is committed but also regenerated by CI; local builds will overwrite it.

## Deployment

Pushes to `master` trigger the GitHub Actions workflow which:
1. Builds with `hugo --gc --minify`
2. Uploads `public/` as a GitHub Pages artifact and deploys it

No manual deploy step is needed.
